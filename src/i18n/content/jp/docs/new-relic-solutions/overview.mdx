ソフトウェアのパフォーマンスを正確に把握することで、問題を特定し、迅速に解決し、改善点を追跡することができます。また、ソフトウェア、カスタマーエクスペリエンス、ビジネス成果をさらに向上させるために、各反復から学びたいと考えていることでしょう。

当社のガイドは、目標に基づいて構築およびカスタマイズできる基盤を提供します。ジャーニーの各ステップで、推奨される重要業績評価指標 (KPI)、ダッシュボード、ハウツー ドキュメントなどを提供します。まだ New Relic をご利用になっておらず、ご質問がある場合は、 [New Relic のセールスにお問い合わせ](https://newrelic.com/contact-sales)ください。

当社の実証済みの方法論、推奨事項、および専門家のガイダンスを使用して、組織はより迅速に行動し、コースを維持し、変革を成功させることができます!ガイドから始めましょう:

* **New Relic をセットアップし**ます。オプションについては[、インストール ドキュメントを参照して](/docs/new-relic-solutions/new-relic-one/install-configure/install-new-relic)ください。
* **プラットフォームについて学ぶ**: 一般的なプラットフォームのエクスペリエンスと機能を理解したいですか?[プラットフォーム UI ドキュメント を参照して](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-one)ください。
* **ベストプラクティスガイド**：[いくつかの主要なNewRelic機能のベストプラクティスを学びます](/docs/new-relic-solutions/best-practices-guides)。
* **オブザーバビリティの成熟度**: オブザーバビリティの成熟[度シリーズ](/docs/new-relic-solutions/observability-maturity/introduction)では、一般的なオブザーバビリティの目標を特定し、それらを達成する方法を説明します。
* **クラウド移行ソリューション**: クラウドに移行しますか?[アプリケーションの異常を作成し](/docs/new-relic-solutions/new-relic-solutions/create-application-baselines)[、アーキテクチャと支出を最適化する](/docs/new-relic-solutions/new-relic-solutions/optimize-cloud-architecture-spend-continuously-improve-your-modern-cloud-environment)方法を学びます。
* **一般的な問題の解決**：[ヘルプを見つけ、診断を実行し、一般的な問題のトラブルシューティングを行います](/docs/new-relic-solutions/solve-common-issues/find-help-use-support-portal)。
